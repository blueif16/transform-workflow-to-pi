OpenClaw Domain Glossary

A compact vocabulary for reading vendor/openclaw without collapsing product terms, runtime terms, and repository boundaries into one pile.

Canonical source: CONTEXT.md | First lesson: OpenClaw Outer Shell

Core Terms

Term Definition Do Not Confuse With
OpenClaw The vendored OpenClaw repository studied first in this learning track. Hermes Agent, generic agent frameworks
Gateway The long-lived control plane for channels, sessions, tools, events, clients, and nodes. Any backend process, any HTTP server
Channel A messaging surface that carries inbound and outbound conversation traffic. Companion app, provider, node
Node A device or headless peer that connects to the Gateway and exposes declared capabilities. Control client, channel adapter
Agent Runtime The embedded execution surface for model discovery, tools, prompts, sessions, and delivery. A standalone external agent framework
Agent Workspace The working directory OpenClaw gives the runtime for tools, context, skills, and bootstrap files. The OpenClaw source checkout
Session A conversation context selected from message origin, lifecycle rules, and isolation policy. Plain chat transcript, UI thread
Plugin A capability package that owns provider, channel, tool, or feature behavior outside core. Core runtime module
Plugin SDK The supported contract plugins use to cross into core without importing internals. Private implementation files

Reading Heuristic

When a file looks important, classify it before reading deeply: operator path, Gateway, agent runtime, session, plugin, SDK contract, or companion surface. This keeps the map small while the repo stays large.
