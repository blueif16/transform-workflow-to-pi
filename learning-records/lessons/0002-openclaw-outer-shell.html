The Smallest Useful Model

The OpenClaw README names the product: a personal AI assistant that runs on your devices. It also says the Gateway is the control plane, not the whole product.

The local Gateway architecture doc sharpens that: one long-lived Gateway owns messaging surfaces, exposes a WebSocket API, accepts clients and nodes, emits events, and serves canvas surfaces.

The agent runtime doc gives the next boundary: OpenClaw has one embedded runtime per Gateway, with its own workspace, bootstrap files, tools, skills, and session store.

Repository Shape

The source checkout is a pnpm workspace: root package, ui, packages/*, and extensions/*. That is declared in pnpm-workspace.yaml.

The published package is named openclaw, exposes the openclaw binary through openclaw.mjs, and publishes many ./plugin-sdk/* subpaths for supported plugin contracts.
