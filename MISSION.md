# Mission: Learn OpenClaw and Hermes Agent

## Why
Build a grounded understanding of the two vendored agent repos so their architecture, vocabulary, and tradeoffs can inform future Pi/OpenClaw/Hermes work without relying on guesses or broad agent theory.

## Success looks like
- Navigate `vendor/openclaw` and `vendor/hermes-agent` from their real entry points, instructions, and package boundaries.
- Explain the core control-plane, session, gateway, plugin, skill, and provider concepts in each repo.
- Compare OpenClaw and Hermes using a shared vocabulary captured in this workspace.
- Produce short lessons and quick-reference documents that cite local files.

## Constraints
- Treat this repository root as the teaching workspace.
- Scope the first learning track to `vendor/openclaw` and `vendor/hermes-agent`.
- Start with OpenClaw before moving into Hermes.
- Keep domain modeling limited to language needed for these two repos.

## Out of scope
- Implementing features in either vendored repo.
- Broad AI-agent theory that is not tied to local source files.
- Domain modeling for projects outside OpenClaw and Hermes Agent.
