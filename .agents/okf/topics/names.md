# Why / how it works (the lifecycle, end to end)
The naming alphabet is two DATA lists: `ADJECTIVES` (hand-authored `adjectives.json`, ~45 baking flavors) and
`PIES` (`pies.json`, NOT hand-authored — DERIVED). The single slug rule `pieSlug` (fold diacritics, expand
ligatures, hyphenate, drop a trailing generic like "pie"/"tart") plus `pieSlugList` (dedupe + sort) is what the
build-time script `generate-pies.mjs` runs over a source CSV to materialize `pies.json` deterministically — so the
word list is auditable, not magic. At run time `generateRunName(existing, rng)` `pick()`s one of each into an
`<adjective>-<pie>` string and re-picks on collision against the already-used names (the `Rng` seam keeps it
deterministic in tests). It is re-exported from the `@piflow/core` public API, and the CLI's `runTemplate` is the
TERMINAL: when `--run` is omitted it mints `runId = generateName(listExistingRuns(landingHome))`, and that id
becomes the run's home directory `.piflow/<wf>/runs/<id>/` and its `status.run`.
